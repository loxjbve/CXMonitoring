`CXMonitoring` is a sleek, mobile-first web dashboard that lets you keep an eye on your AI coding agent (Codex) while you step away from your desk. It securely streams live events, command executions, and token usage straight to your browser over your local network.

## ✨ Features

- 🕵️ **Auto-Detection**: Instantly detects the latest active VS Code Codex thread.
- ⚡ **Live Streaming**: Replays the current rollout file and streams new JSONL events to the browser over Server-Sent Events (SSE).
- 📱 **Mobile-First UI**: Features a beautiful, responsive, drawer-based layout designed for seamless monitoring on mobile devices.
- 🔒 **Read-Only**: Strictly monitors your agent without modifying the OpenAI extension or writing back into the Codex state.
- 📊 **Rich Metrics**: Displays the current task title, latest agent progress, shell commands, tool activities, and detailed token usage.

## 🛠️ How It Works

The dashboard tracks your agent's state by reading local files generated by the VS Code extension:
- **Active Thread Metadata**: `%USERPROFILE%\.codex\state_5.sqlite`
- **Live Event Stream**: `%USERPROFILE%\.codex\sessions\...\rollout-*.jsonl`

## 🚀 Quick Start

### 1. Install Dependencies

You can run the server directly by installing the required packages:

```bash
python -m pip install fastapi uvicorn httpx
```

*Alternatively, install the project package in editable mode:*

```bash
python -m pip install -e .
```

### 2. Start the Server

Run the standalone script:

```bash
python run_server.py
```

*Or run it as a module:*

```bash
python -m cxmonitoring
```

### 3. Open the Dashboard

Navigate to one of the following URLs in your browser:
- Local: `http://127.0.0.1:3180`
- LAN (from your phone): `http://<your-lan-ip>:3180`

## ⚙️ Configuration (Environment Variables)

You can customize the server behavior using the following environment variables:

| Variable | Description | Default |
| :--- | :--- | :--- |
| `CXMONITORING_CODEX_HOME` | Override the default Codex home directory | *(Auto-detected)* |
| `CXMONITORING_HOST` | Bind host address | `0.0.0.0` |
| `CXMONITORING_PORT` | Port to listen on | `3180` |
| `CXMONITORING_THREAD_POLL` | Thread refresh interval (seconds) | `1.0` |
| `CXMONITORING_ROLLOUT_POLL` | Rollout tail interval (seconds) | `0.3` |

## 🧪 Running Tests

Ensure everything is working correctly by running the test suite:

```bash
python -m unittest discover -s tests
```

## ⚠️ Security Notice

By default, this server binds to `0.0.0.0:3180` without authentication. Anyone on your local network (LAN) can view your current task, recent prompts, tool summaries, and command outputs. Please use responsibly on trusted networks.
